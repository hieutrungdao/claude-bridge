## What This Task Does
Wire a user-facing CLI — `bridge-cli wiki ingest [--agent NAME] [--dry-run]` — on top of the M24.T2 runner. Supports per-project opt-out via a `.bridgewiki-ignore` marker file. Prints progress, propagates exit codes, and in `--dry-run` mode previews what ingest would do without invoking `claude -p` (free, safe).

## Key Architecture Constraints
- **Follow the `daemon` pattern** in cli.py: `wiki` is a subcommand group with its own nested parser, dispatched via a special-case `_cmd_wiki(args)` function outside the `COMMANDS` dict. Keeps the main dispatch loop unchanged.
- **`.bridgewiki-ignore` is CLI-level, not collector-level.** Filtering lives in the CLI so `wiki.collect_sources()` stays a pure fan-out over registered agents (M23.T2's contract).
- **`--dry-run` never spawns a subprocess and never writes a `wiki_operations` row.** It only reads: agents, sources, mtimes. Cost = 0, safe to run anywhere.
- **Progress output to stdout, errors to stderr** (repo-wide convention per `.claude/rules/code-style.md`).
- **Exit codes**: 0 on success or clean skip; non-zero propagates the `claude -p` exit code verbatim so CI pipelines can detect failed synthesis.
- **Stdlib only.** argparse subparsers + existing wiki module calls.

## Design

### CLI surface

```
$ bridge-cli wiki --help
usage: bridge-cli wiki [-h] {ingest} ...

$ bridge-cli wiki ingest --help
usage: bridge-cli wiki ingest [-h] [--agent NAME] [--dry-run]

options:
  --agent NAME   narrow to a single agent
  --dry-run      print what would happen without invoking claude -p
```

Future `wiki query` (M25) and `wiki lint` (M26) slot into the same group as additional subparsers under `wsub`.

### Dispatch in cli.py

Following the `daemon` pattern:

```python
# in build_parser() — alongside the daemon block
w = sub.add_parser("wiki", help="Compounding wiki memory operations")
wsub = w.add_subparsers(dest="wiki_cmd", required=True)

wp = wsub.add_parser("ingest", help="Synthesize agent memories into the wiki")
wp.add_argument("--agent", metavar="NAME", default=None,
                help="narrow to a single agent")
wp.add_argument("--dry-run", action="store_true",
                help="preview without invoking claude")

# in main() — alongside the other if-branches
if args.command == "wiki":
    sys.exit(_cmd_wiki(args))
```

### `_cmd_wiki` handler

```python
def _cmd_wiki(args) -> int:
    if args.wiki_cmd == "ingest":
        return _cmd_wiki_ingest(args)
    print(f"Error: unknown wiki subcommand: {args.wiki_cmd}", file=sys.stderr)
    return 1
```

### `_cmd_wiki_ingest` handler

1. Collect sources via `wiki.collect_sources(agent_filter=args.agent)`.
2. Filter out any source whose `project_dir` contains a `.bridgewiki-ignore` file. Report the skipped count.
3. If no sources remain, print "No sources to ingest." and exit 0.
4. If `args.dry_run`: print the agents and their `source_mtime` values, print the would-be prompt length (informational only), exit 0. Never call `wiki.ingest()`.
5. Otherwise: call `wiki.ingest(agent_filter=args.agent)` (which re-reads sources + applies the same filter via the filter helper). Print:
   - On `skipped=True`: `[ingest] Skipped — sources unchanged since last successful run`, exit 0
   - On `exit_code=0`: `[ingest] Done — <N> sources, <M> pages changed, cost=$X.XXXX, duration=Ymms`, exit 0
   - On `exit_code != 0`: print `[ingest] Failed — exit {code}` to stderr, dump `stderr` field, exit with that code

### `.bridgewiki-ignore` filter

```python
def _filter_ignored(sources: list) -> tuple[list, list[str]]:
    """Split sources into (kept, ignored_agent_names)."""
    kept = []
    ignored = []
    for rec in sources:
        marker = os.path.join(rec["project_dir"], ".bridgewiki-ignore")
        if os.path.isfile(os.path.expanduser(marker)):
            ignored.append(rec["agent"])
        else:
            kept.append(rec)
    return kept, ignored
```

To apply this filter to the actual ingest subprocess call (not just the dry-run preview), extend `wiki.ingest()` with a `source_filter: Callable | None = None` parameter. The CLI passes a closure that delegates to `_filter_ignored`; internal callers can pass None for the existing pure behavior. This is an additive, backward-compatible API change — existing M24.T2 tests remain green.

## Test Plan

**New file:** `tests/test_wiki_cli.py` — invokes `main()` with `sys.argv` patched, captures stdout/stderr via pytest's `capsys`.

**Happy path:**
- `bridge-cli wiki --help` exits 0 and lists `ingest`
- `bridge-cli wiki ingest --help` exits 0 and lists `--agent` and `--dry-run`
- With 1 agent + memory, mocked subprocess → exits 0, stdout contains `Done`, cost and pages

**Dry run:**
- `--dry-run` never invokes `subprocess.run` (assertion via monkeypatch)
- `--dry-run` never writes a `wiki_operations` row
- Output lists the agent names that would be ingested

**No sources:**
- Empty DB → exits 0, stdout contains `No sources to ingest`

**Agent filter:**
- `--agent backend` narrows to backend only; stdout reflects 1 source

**`.bridgewiki-ignore`:**
- Drop a `.bridgewiki-ignore` file into one agent's project_dir → that agent is excluded
- Output mentions the ignored agent count

**Failure propagation:**
- Mocked subprocess returns exit_code=2 → CLI exits 2
- stderr output contains the captured stderr text

**Idempotent skip:**
- Second ingest immediately after a successful first → stdout contains `Skipped`, exit 0

**Test harness for CLI:**
- `monkeypatch.setattr(sys, "argv", [...])` + direct call to `main()` from `claude_bridge.cli`
- Use `pytest.raises(SystemExit)` to catch `sys.exit`; check `.value.code`
