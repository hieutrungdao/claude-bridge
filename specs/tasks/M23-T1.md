## What This Task Does
Create `src/claude_bridge/wiki.py` with a `wiki_home()` helper that returns the wiki directory path (`~/.claude-bridge/wiki/`) and lazily initializes it with `schema.md`, `index.md`, and an empty `log.md` on first access. Ships a bundled schema template inside the package so `pip install`ed users get the full operating manual. Includes a `safe_write()` helper that rejects writes resolving outside the wiki directory.

## Key Architecture Constraints
- Wiki lives under `~/.claude-bridge/wiki/` (respects `CLAUDE_BRIDGE_HOME` env var)
- Bridge NEVER writes to `~/.claude/projects/*/memory/` — verified in tests
- Init is idempotent: re-running never overwrites existing `schema.md`, `index.md`, or `log.md`
- All path operations use `Path.resolve()` before the boundary check (defeats symlink attacks and `..` traversal)
- Stdlib only — no external dependencies
- Type hints on all public functions; errors to stderr; exit code 0 = success

## Design

### Public API

```python
# src/claude_bridge/wiki.py

from pathlib import Path

def wiki_home() -> Path:
    """Return the wiki home directory, initializing it on first access.

    Creates ~/.claude-bridge/wiki/ and seeds schema.md, index.md, log.md
    if they don't exist. Idempotent.
    """

def safe_write(relative_path: str, content: str, *, overwrite: bool = False) -> Path:
    """Write content to a file inside the wiki home.

    Raises ValueError if relative_path resolves outside wiki_home().
    Raises FileExistsError if the file exists and overwrite is False.
    Returns the absolute path written.
    """

def safe_read(relative_path: str) -> str:
    """Read a file inside the wiki home. Raises ValueError if outside."""
```

### Boundary Guard

```python
def _assert_inside_wiki(path: Path) -> Path:
    wiki = wiki_home().resolve()
    resolved = (wiki / path).resolve() if not path.is_absolute() else path.resolve()
    try:
        resolved.relative_to(wiki)
    except ValueError:
        raise ValueError(
            f"Path {path!r} resolves outside wiki home {wiki!r}"
        )
    return resolved
```

Every `safe_write` / `safe_read` routes through this. Rejects `..`, symlinks pointing outside, and absolute paths not rooted in the wiki.

## Test Plan

**Happy path:**
- `wiki_home()` returns `<bridge_home>/wiki/` and creates the directory
- First call seeds `schema.md`, `index.md`, `log.md`
- Seeded `schema.md` contains "Karpathy" attribution, "Page Template", "Ingest", "Query", "Lint"
- Seeded `index.md` has today's date in `Last updated:` line
- Seeded `log.md` is short and append-ready (contains header, no entries)
- `safe_write("notes/foo.md", "x")` creates `<wiki>/notes/foo.md` with content "x"
- `safe_read("schema.md")` returns the seeded content

**Idempotency:**
- Calling `wiki_home()` twice does not rewrite `schema.md` even if its content was edited between calls
- Calling `wiki_home()` twice does not rewrite `index.md` or `log.md`
- User-edited `schema.md` survives a second init

**Edge cases:**
- `CLAUDE_BRIDGE_HOME` override redirects wiki to the custom path
- `safe_write(..., overwrite=False)` on existing file raises `FileExistsError`
- `safe_write(..., overwrite=True)` replaces content
- Parent directories auto-created for nested writes (`safe_write("entities/api.md", ...)`)

**Boundary / errors:**
- `safe_write("../outside.md", "x")` raises `ValueError`
- `safe_write("/tmp/escape.md", "x")` raises `ValueError`
- `safe_read("../../../../etc/passwd")` raises `ValueError`
- Symlink inside wiki pointing to `/tmp` → write via symlink raises `ValueError` (resolve before check)
- Never writes under `~/.claude/projects/` — test uses a monkey-patched `open` that records every path opened for writing and asserts none start with `~/.claude/projects/`

**Uses `tmp_path` + `CLAUDE_BRIDGE_HOME`:**
All tests set `monkeypatch.setenv("CLAUDE_BRIDGE_HOME", str(tmp_path))` so no test touches the real home directory.
