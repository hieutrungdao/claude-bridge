# M23.T2: Source Collector

## What This Task Does
Add `collect_sources()` to `wiki.py`. For every registered agent (optionally filtered by name), read its Auto Memory via `memory.read_memory()`, compute a `source_mtime` (max mtime across MEMORY.md + topic files), and return a list of source records. This is the input to the future ingest pipeline (M24) — it packages everything the synthesis prompt needs about each agent's current memory state.

## Key Architecture Constraints
- Bridge reads Auto Memory read-only. Collector must not write anywhere under `~/.claude/projects/*/memory/` — verified by monkey-patched `open` test.
- Agents with no Auto Memory yet return a valid record with `found=False` (not an error, not a skip). Ingest needs to see them so it can emit a "no sources yet" log entry.
- `source_mtime` is None when `found=False`. When present, it is a POSIX timestamp (float) representing the newest change across all memory files; future ingest uses this to short-circuit when nothing has changed since last run.
- Supports dependency injection: `collect_sources(db=...)` lets tests pass a mock or an alternate `BridgeDB` instance. When `db` is None, a default one is created and closed inside the function.
- Stdlib only. Matches `.claude/rules/code-style.md`: no new classes (uses a `TypedDict` for shape only).

## Design

### Public API

```python
# src/claude_bridge/wiki.py

from typing import TypedDict

class SourceRecord(TypedDict):
    agent: str            # agent name
    project_dir: str      # absolute project path
    session_id: str       # "{agent}--{project_basename}"
    found: bool           # whether Auto Memory dir exists
    memory_dir: str | None  # absolute path to memory dir, None if not found
    main_memory: str      # MEMORY.md content (empty string if none)
    topics: list[dict]    # [{"name": ..., "content": ...}]
    source_mtime: float | None  # max mtime across *.md files, None if none

def collect_sources(
    agent_filter: str | None = None,
    db: "BridgeDB | None" = None,
) -> list[SourceRecord]:
    """Collect Auto Memory sources for all registered agents.

    agent_filter narrows to one agent by name. db can be injected for
    tests; when None, a default BridgeDB() is created and closed here.
    """
```

### Behavior

1. If `db` is None, construct `BridgeDB()` and track that we own it (must close).
2. Query `db.list_agents()` → rows with `name`, `project_dir`, `session_id`.
3. If `agent_filter` is set, drop rows where `name != agent_filter`.
4. For each row, call `memory.read_memory(project_dir)`.
5. Compute `source_mtime` via a small helper `_max_mtime(memory_dir)` — iterates `*.md` files, returns max `os.path.getmtime`, None if no memory_dir or empty.
6. Build and append a `SourceRecord`.
7. Close owned DB.
8. Return the list in the same order `list_agents()` returned.

### What the record does NOT include

- No task transcripts (out of scope for Phase 6 per architecture doc open question — defer to v2 if needed).
- No diffs between this ingest and the last. Staleness checking lives in the ingest command (M24), not here.

## Test Plan

**Happy path:**
- `collect_sources()` with no agents returns `[]`
- With 2 agents + real Auto Memory files, returns 2 records with correct content and mtimes
- `agent_filter="backend"` returns only the backend record
- `agent_filter` for a non-existent agent returns `[]`
- Records preserve order from `list_agents()`

**Empty/missing Auto Memory:**
- Agent exists in DB but has no `~/.claude/projects/<...>/memory/` directory → record has `found=False`, `main_memory=""`, `topics=[]`, `source_mtime=None`, `memory_dir=None`
- Memory directory exists but is empty → `found=True`, `source_mtime=None`

**mtime correctness:**
- `source_mtime` equals the max of `MEMORY.md` mtime and topic-file mtimes
- Skips non-`.md` files when computing mtime
- A freshly written topic file pushes the mtime forward

**Boundary / ownership:**
- Never writes anywhere under `~/.claude/projects/` — monkey-patched `open` records all writes and asserts none hit that prefix (same pattern as M23.T1)

**Dependency injection:**
- `collect_sources(db=mock_db)` does not construct a default BridgeDB
- `collect_sources()` with no `db` arg works against `CLAUDE_BRIDGE_HOME` tmp_path (real BridgeDB, empty, returns [])

**Errors are graceful:**
- Memory directory exists but a topic file is unreadable → either collector raises a specific exception or skips the file with a stderr message (decide at impl time, document in tests)

## Gaps in Existing Code
- No `collect_sources` or equivalent exists in `wiki.py`
- `memory.read_memory()` returns content but not mtime — this task computes mtime separately inside `wiki.py` rather than extending `memory.py`, keeping concerns separate (wiki.py: coordinate; memory.py: read)
- No `SourceRecord` shape anywhere — introduced here as `TypedDict`; later ingest/query tasks will import from `wiki.py`
