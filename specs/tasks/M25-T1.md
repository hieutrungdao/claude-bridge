## What This Task Does
A pure-function retriever: given a natural-language question, tokenize, score every wiki entity page by token overlap with title / headers / body (weighted 3 / 2 / 1), return the top-K. Stdlib only — no LLM, no subprocess, no vector store. This is the "fetch candidates" half of M25's query pipeline; M25.T2 layers synthesis on top.

## Key Architecture Constraints
- **Stdlib only.** `re` for tokenization, `pathlib` for enumeration, `os.path.getmtime` for tiebreaker. No NLP libs, no embeddings.
- **Pure read, pure return.** No writes to wiki or DB. Safe to call as often as needed.
- **Exclude operational pages.** `schema.md`, `index.md`, `log.md` never appear in results — they're wiki infrastructure, not knowledge.
- **Weighted scoring.** Title matches worth 3x, header matches 2x, body matches 1x. Prevents "the word appears once in a footnote" pages from outranking "the page is literally about that topic."
- **Tie-break on recency.** When scores are equal, newer `Last updated` wins — biases answers toward fresher synthesis.
- **Unicode-aware.** Questions and wiki content can be non-ASCII (e.g., Vietnamese). `\w+` tokenization handles this natively in Python 3 re.
- **Stopwords minimal.** Small hand-curated English set; nothing fancy like stemming or language detection. Karpathy's guidance: "index file may be all you need" — start simple, escalate only if results are poor.

### Tokenizer

```python
STOPWORDS = frozenset({
    "a", "an", "and", "any", "are", "as", "at", "be", "by", "do", "does",
    "for", "from", "has", "have", "how", "if", "in", "is", "it", "its",
    "of", "on", "or", "our", "that", "the", "their", "this", "to", "was",
    "we", "what", "when", "where", "which", "who", "why", "will", "with",
    "would", "you", "your",
})

def _tokenize(text: str) -> set[str]:
    raw = re.findall(r"\w+", text.lower(), flags=re.UNICODE)
    return {t for t in raw if len(t) >= 2 and t not in STOPWORDS}
```

### Scoring

Parse each page into three regions:

- **Title**: first `# ...` line (H1). Fallback: filename stem with dashes → spaces.
- **Headers**: all `##`, `###`, `####` lines (H2+).
- **Body**: everything else.

For each question token, compute `weight = 3*in_title + 2*in_headers + 1*in_body` (booleans summed). Total score = sum over all question tokens. Zero if no overlap.

### Excerpt selection

Find the header line (any level) that contains the most question tokens. Return the first 2 non-empty lines immediately after that header. If no header matched, use the title line + first non-empty body line.

### Exclusions

Skip files named `schema.md`, `index.md`, `log.md`. Also skip anything in subdirectories (future-proof for Phase 6 v2 sub-trees; for now wiki is flat).

## Test Plan

**Empty state:**
- No entity pages → `[]`
- Wiki has only schema/index/log → `[]`
- Empty question (after stopword stripping) → `[]`

**Happy path:**
- Single entity page matches exactly → returned with `score > 0`, correct path, title, excerpt
- Multiple pages match → returned in score-descending order
- `top_k=1` caps results at 1

**Scoring weights:**
- Page with token in title outranks page with same token only in body
- Page with token in H2 outranks page with same token only in body

**Tiebreak:**
- Two pages with identical score → the one with newer `Last updated` (via `os.path.getmtime`) comes first

**Excerpt:**
- Excerpt is the first 2 non-empty lines under the best-matching header
- When no header matched, falls back to title + first body line

**Exclusions:**
- `schema.md` never appears even when the question matches its content
- `index.md` and `log.md` never appear

**Unicode:**
- Vietnamese question `"làm thế nào để xử lý X"` tokenizes and matches a Vietnamese wiki page with the same content

**Stopwords:**
- Question `"the api"` behaves the same as `"api"` (the dropped)
