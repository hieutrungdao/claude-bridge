## What This Task Does
Implement `wiki.ingest()` — the orchestrator that pulls together the three pieces M23–M24.T1 built: `collect_sources` gathers per-agent Auto Memory, the ingest prompt tells `claude -p` how to synthesize, and a new `wiki_operations` SQLite table records every run with cost and duration. Idempotent when sources haven't changed since the last successful ingest.

## Key Architecture Constraints
- **Stdlib + existing deps only.** Uses `subprocess.run` (not `Popen` — ingest is synchronous, unlike agent dispatch) and `json` to parse `claude -p --output-format json`.
- **Idempotency by source mtime.** `wiki_operations` stores `last_source_mtime` (float) at ingest time. On a re-run, the runner compares the current max source mtime against the last successful ingest's `last_source_mtime`; if unchanged, returns a `skipped=True` result with no subprocess call and no row written.
- **Subprocess pattern matches `dispatcher.py`:** `claude -p <prompt> --output-format json --dangerously-skip-permissions`, `cwd=wiki_home()`. No `--agent` flag (ingest is ad-hoc, not an agent task).
- **Bounded by wiki home at cwd + prompt rules.** Claude's Edit/Write paths default to `cwd`; the M24.T1 prompt explicitly forbids escaping. The boundary is belt-and-suspenders: code-level (`safe_write` for Bridge-owned writes) + prompt-level (for Claude-owned writes during ingest).
- **Failure-safe logging.** A row is written to `wiki_operations` even on non-zero exit so users can audit failed ingests. `exit_code` column distinguishes success from failure; `stderr` is captured.
- **Tests mock `subprocess.run`.** One opt-in `@pytest.mark.integration` test invokes the real `claude` CLI; skipped unless `-m integration` is passed.

## Design

### Public API

```python
# src/claude_bridge/wiki.py

class IngestResult(TypedDict):
    skipped: bool                  # True if sources unchanged since last ingest
    sources_count: int             # agents processed (0 when skipped)
    pages_changed: list[str]       # wiki-relative paths touched by Claude
    cost_usd: float
    duration_ms: int
    exit_code: int                 # 0 = success
    stderr: str                    # empty on success

def ingest(
    agent_filter: str | None = None,
    db: "BridgeDB | None" = None,
) -> IngestResult:
    """Run one ingest: collect sources → synthesize via claude -p → record run.

    Skipped (no subprocess, no row) when max source mtime has not advanced
    since the last successful ingest.
    """
```

### Schema addition in `db.py`

```sql
CREATE TABLE IF NOT EXISTS wiki_operations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    operation TEXT NOT NULL,           -- 'INGEST' | 'QUERY' | 'LINT'
    started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    duration_ms INTEGER,
    cost_usd REAL DEFAULT 0.0,
    sources_count INTEGER,
    pages_changed TEXT,                 -- JSON array of wiki-relative paths
    agent_filter TEXT,
    exit_code INTEGER,
    stderr TEXT,
    last_source_mtime REAL              -- max mtime at time of this run
);
```

Keeping the schema in `db.py` is load-bearing for consistency — every other table lives there. `wiki.py` queries the table via `BridgeDB.conn.execute(...)` without adding new helper methods to `BridgeDB`.

### Prompt assembly (runtime-only, lives in wiki.py)

```
{static template from wiki_ingest.md}

---

## Current Schema
{contents of wiki/schema.md}

## Current Index
{contents of wiki/index.md}

## Existing Entity Pages
{for each wiki/*.md except schema/index/log:}
### {filename}
```
{first 120 lines}
```

## New Sources
```json
[
  {"agent": "backend", "project_dir": "/projects/api", "session_id": "...",
   "found": true, "main_memory": "...", "topics": [...]},
  ...
]
```

---

Now synthesize. Begin by calling Read on any pages you need to update.
```

Helper: `_assemble_ingest_prompt(sources, wiki_state) -> str`. Pure string work, trivially unit-testable.

### Pages-changed detection

1. **Pre-snapshot** — `{path: mtime}` for every `wiki/*.md` immediately before the subprocess call.
2. **Run subprocess.**
3. **Post-snapshot** — same.
4. `pages_changed` = files with new mtimes OR files that didn't exist pre.

Excludes `schema.md` and `log.md` touches from counting as "entity page" changes? No — include any file the worker touched, including `log.md` (which it's instructed to append to). `pages_changed` is a transparency column, not a semantic one.

### Idempotency check

```python
last_mtime = db.conn.execute(
    "SELECT MAX(last_source_mtime) FROM wiki_operations "
    "WHERE operation = 'INGEST' AND exit_code = 0"
).fetchone()[0]
current_max = max((r["source_mtime"] or 0.0) for r in sources) if sources else 0.0
if last_mtime is not None and current_max <= last_mtime:
    return {"skipped": True, ...}
```

### Cost parsing

`claude -p --output-format json` emits a single JSON object on stdout with at least `total_cost_usd` and `duration_ms`. Runner parses with `json.loads(stdout)`, extracts both; falls back to 0/0 on parse error (rare — logged to stderr column).

## Test Plan

**Happy path / subprocess invocation:**
- With 1 agent + memory, `ingest()` calls `subprocess.run` once with:
  - `cwd == str(wiki_home())`
  - argv starts with `["claude", "-p", <prompt>]`
  - argv contains `"--output-format"` `"json"`
- Returns `IngestResult` with `skipped=False`, `sources_count=1`
- `wiki_operations` row inserted with `operation='INGEST'`, `exit_code=0`, correct `cost_usd`, `duration_ms`
- `pages_changed` lists files actually modified by the mocked worker

**Idempotency:**
- Second ingest immediately after a successful first → `skipped=True`, no subprocess call, no new row
- Second ingest after a source mtime advances → subprocess called again, new row written
- First-ever run (no prior row) always runs

**Empty state:**
- No agents registered → `skipped=True`, `sources_count=0`, no subprocess
- Agents registered but all have `found=False` → still proceeds (first run); runner emits a "no sources found" log line via Claude

**Agent filter:**
- `agent_filter="backend"` narrows the sources passed into the prompt; only backend's data appears in the `## New Sources` section

**Failure handling:**
- Subprocess returns exit_code=2 → `IngestResult.exit_code=2`, `stderr` populated, `wiki_operations` row written with `exit_code=2`
- Invalid JSON on stdout → `cost_usd=0.0`, `duration_ms=0`, `stderr` contains a parse-error note; row still written

**Prompt assembly (pure):**
- `_assemble_ingest_prompt(...)` output contains the static template, the current schema, and the source JSON block
- Order is deterministic given deterministic inputs

**Boundary invariant:**
- Unit-level: monkey-patched `subprocess.run` never receives a `cwd` outside wiki home
- Unit-level: `ingest()` itself does not `open()` anything under `~/.claude/projects/` (same watchdog pattern as M23.T2)

**Integration (opt-in):**
- `@pytest.mark.integration` test that calls real `claude -p` end-to-end against a small synthetic agent setup. Skipped unless `pytest -m integration` is passed. Documents how to run.

## Gaps in Existing Code
- No `wiki_operations` table exists
- No `subprocess.run` callers in the codebase yet (dispatcher.py uses `Popen` for async); this task establishes the synchronous pattern that M25/M26 will reuse
