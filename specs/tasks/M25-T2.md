## What This Task Does
Ship `wiki.query(question)` — the top-level query API. Calls `retrieve()` to fetch candidate pages, hands them to `claude -p` with a synthesis prompt that requires inline `[Source: page.md]` citations, returns a structured `QueryResult` and writes a `wiki_operations` row for cost/duration audit. Mirrors M24.T2's subprocess shape so future M25.T3 (CLI + MCP) can wrap the same function.

## Key Architecture Constraints
- **Stdlib only** — `subprocess.run`, `json`, `re` for citation extraction.
- **Subprocess pattern matches M24.T2**: `claude -p <prompt> --output-format json --dangerously-skip-permissions`, `cwd=wiki_home()`, 5-minute timeout (shorter than ingest — queries should be fast).
- **Read-only by default for v1.** The phase doc allows Claude to write novel syntheses back into wiki pages; this task ships query as **pure read** (no writes, no pages_changed detection). Writeback is deferred to a future task so M25 can ship without requiring the harder "when is a synthesis novel?" judgment.
- **Citations are contractual.** Prompt requires `[Source: <filename>]` format. Runner extracts citations with a regex and returns them as `sources_cited` — caller (CLI or MCP tool) decides how to render.
- **Empty retrieve → structured empty answer.** No subprocess call when `retrieve()` returns `[]`. Returns a `QueryResult` with a helpful message and `skipped=True`-equivalent fields, no wiki_operations row written.
- **SHA-pinned prompt** — same pattern as M24.T1.

## Design

### Public API

```python
class QueryResult(TypedDict):
    answer: str                # synthesized answer with inline citations
    sources_cited: list[str]   # page paths mentioned via [Source: ...]
    pages_retrieved: list[str] # paths passed into claude (before citing)
    cost_usd: float
    duration_ms: int
    exit_code: int
    stderr: str
    empty: bool                # True when retrieve() returned []

def query(question: str, top_k: int = 5, db: "BridgeDB | None" = None) -> QueryResult:
    """Answer a question from wiki content, with citations."""
```

### Prompt file: `src/claude_bridge/prompts/wiki_query.md`

Static preamble (snapshot-pinned):

- Role preamble — "You answer questions using only the provided wiki pages."
- Tools — **none needed.** Query is pure text generation. Prompt explicitly says: "Do not invoke any tool. Return only your answer text on stdout."
- Citation format — `[Source: <filename>]` inline after each claim
- Fidelity rule — if the retrieved pages don't answer the question, say so; never hallucinate
- Format — short answer (2-5 sentences), citations inline
- 2 good examples + 2 bad examples (matches ingest prompt structure)

### Runtime assembly

```
<static template>

---

## Question
<user's question>

## Retrieved Pages
### <filename 1>
```
<full page content>
```
### <filename 2>
...

---

Answer now. Remember: inline [Source: filename.md] citations required.
```

### Subprocess + parsing

- `claude -p <full_prompt> --output-format json --dangerously-skip-permissions`, `cwd=wiki_home()`, timeout 300s
- Parse stdout JSON → `result` field contains the answer text; `total_cost_usd` + `duration_ms` as before
- Extract citations: `re.findall(r"\[Source:\s*([^\]]+?\.md)\s*\]", answer)` → list of unique filenames, preserving first-occurrence order

### Failure handling

- Non-zero exit → `QueryResult` with `exit_code != 0`, `answer=""`, `stderr` populated, row still written (same audit guarantee as ingest)
- Malformed JSON → `cost_usd=0.0`, `answer` is the raw stdout (best-effort)
- Retrieve empty → no subprocess call, `empty=True`, helpful `answer` text, no row

### Prompt template (`test_wiki_query_prompt.py`)

- Mentions fidelity/hallucination warning
- ≥2 good + ≥2 bad example markers (✓ / ✗)
- SHA-256 matches pinned `EXPECTED_SHA`

### Runtime (`test_wiki_query.py`)

- **Empty retrieve:** no entity pages → `empty=True`, no subprocess, no DB row
- **Happy path with mock:** retrieve → subprocess → answer + citations parsed; `pages_retrieved` lists candidate paths; `cost_usd` + `duration_ms` from JSON; `wiki_operations` row with `operation='QUERY'`
- **Citation extraction:** answer containing `"Token bucket [Source: api.md]. Per-key [Source: api.md]."` → `sources_cited == ["api.md"]` (deduped, order preserved)
- **Multiple citations:** `"... [Source: auth.md]. And [Source: sessions.md]."` → `["auth.md", "sessions.md"]`
- **Failure path:** subprocess non-zero → `exit_code != 0`, `stderr` preserved, row still written
- **Never writes to wiki:** monkey-patched `open` records writes — none outside bridge_home (query is read-only)
- **Top-k propagation:** `query(q, top_k=3)` calls `retrieve(q, top_k=3)` (verified by patching retrieve)
- **Integration:** one `@pytest.mark.integration` test, gated on `CLAUDE_BRIDGE_RUN_INTEGRATION`
