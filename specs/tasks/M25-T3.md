## What This Task Does
Expose `wiki.query()` on two surfaces: (1) the CLI, via `bridge-cli wiki query "<question>"`, and (2) the MCP server, via a `wiki_query(question, top_k=5)` tool the Bridge Bot can call from Telegram sessions. This is the task that unblocks Telegram integration â€” after it lands, a user can ask the bot a knowledge question and the bot answers from the wiki with citations, without shell-outs.

## Key Architecture Constraints
- **Symmetry with existing MCP tools.** Tool business logic lives in `mcp_tools.py`; `mcp_server.py` only wraps it with `@server.tool()`. Matches `tool_dispatch`, `tool_agents`, etc.
- **JSON-returning MCP tool.** Result is a JSON string (not dict) â€” matches existing convention for all `bridge_*` tools. Consumers parse with `json.loads`.
- **Input validation at the tool layer.** Empty/whitespace-only `question` â†’ returns `{"error": "question cannot be empty"}` JSON. Non-int `top_k` is handled by type coercion.
- **CLI output format** (from acceptance criteria): answer first on stdout, then `Sources:` list, then the cost line. Machine-readable `--json` flag also available for scripting.
- **Registered in `TOOL_NAMES`** â€” the existing test harness asserts every server tool appears in this list.
- **MCP test pattern**: call `mcp_tools.tool_wiki_query` directly with an injected DB. The FastMCP `@tool()` wrapper is trivial and already covered by the existing `TOOL_NAMES` presence check.

### MCP tool (mcp_tools.py)

```python
def tool_wiki_query(db: BridgeDB, question: str, top_k: int = 5) -> str:
    if not question or not question.strip():
        return json.dumps({"error": "question cannot be empty"})
    from . import wiki
    result = wiki.query(question.strip(), top_k=top_k, db=db)
    return json.dumps({
        "answer": result["answer"],
        "sources_cited": result["sources_cited"],
        "pages_retrieved": result["pages_retrieved"],
        "cost_usd": result["cost_usd"],
        "duration_ms": result["duration_ms"],
        "empty": result["empty"],
        "exit_code": result["exit_code"],
    })
```

### Server registration (mcp_server.py)

```python
@server.tool()
def wiki_query(question: str, top_k: int = 5) -> str:
    """Answer a question from the Bridge Wiki with inline citations.

    Args:
        question: The question to answer (non-empty).
        top_k: Max wiki pages to retrieve and synthesize from (default 5).

    Returns a JSON string with keys:
        answer           â€” synthesized answer with [Source: foo.md] citations
        sources_cited    â€” unique page names referenced in answer
        pages_retrieved  â€” candidate pages the retriever picked
        cost_usd         â€” synthesis cost
        duration_ms
        empty            â€” True if the wiki has no matching content
        exit_code        â€” 0 on success
    """
    return mcp_tools.tool_wiki_query(_db(), question, top_k)
```
